# 生产化与 UI 第二阶段重构设计

## 目标

完成当前重构路线中的前两个阶段：补齐存储、日志和通信诊断的生产运行能力，并继续拆分通信页、建设可持久化的系统设置页。

## 方案选择

采用增量兼容方案。现有协议、事件总线、ViewModel 和格式 helper 保持不变，在已有边界上增加小型服务与组件。相比一次性重写存储流水线或引入新的配置框架，该方案改动范围更小，也能继续复用现有 186 项测试。

## 第一阶段：生产化加固

### 存储会话

`LocalImageStorageBackend` 增加显式会话配置。会话启动时创建日期分层目录并写入 `.IMI` 索引；帧请求由会话命名规则生成 `.raw` 或 `.bmp` 文件名。BMP 使用现有 legacy BMP 编码器，RAW 保持现有格式。后端记录已写入、拒绝和失败数量，并发布 `StorageWriteErrorEvent`。

`TrackDataStorageBackend` 增加会话级输出路径。默认仍兼容 `track_data.txt`；配置会话后写入 `GAE/<会话名>.GAE`。写目录、打开文件或写流失败时发布同一类错误事件。

### 滚动日志

配置新增日志路径、单文件大小、保留文件数和启用开关。`Logger` 使用 `rotating_file_sink_mt`，重复配置时替换旧文件 sink，保留控制台和 EventBus sink。应用加载配置后启用日志，失败只记录到现有 sink，不阻止 UI 启动。

### 诊断聚合

`ErrorDiagnostics` 订阅网络发送、串口帧、串口解码和存储写入错误事件，并更新对应诊断位。成功启动服务时通信状态恢复；存储成功写入后存储状态恢复。状态更新与 UDP 周期发送共用受锁保护的快照。

## 第二阶段：UI 组件化

### 通信页

将当前 400 多行通信页拆为串口通道面板、串口命令面板和 UDP 端点面板。`MainWindow` 只负责创建面板并放入页面，面板各自持有编辑器状态和刷新连接。

### 设置页

新增 `SettingsViewModel`，负责读取路径、日志和基础光学配置，校验输入并调用 `Config::save()`。设置页提供对应表单、保存按钮和结果状态，不直接访问 JSON。

## 错误处理与兼容性

- 未配置会话时保留原有相对路径和 `track_data.txt` 行为。
- 文件系统错误通过 `std::expected` 或事件报告，不在 worker 线程抛出异常。
- 配置缺少新字段时使用保守默认值，旧配置文件仍可加载。
- UI 保存失败显示错误文本，成功后刷新内存配置并持久化 JSON。

## 测试

- 存储：会话目录、IMI、BMP/RAW、GAE、写失败事件和统计。
- 日志：滚动配置、重复配置和配置持久化。
- 诊断：各错误事件到诊断位的映射。
- UI：三个通信面板可构造，设置 ViewModel 加载、校验和保存。
- 最终运行 MSVC Debug 全量构建与 CTest。
